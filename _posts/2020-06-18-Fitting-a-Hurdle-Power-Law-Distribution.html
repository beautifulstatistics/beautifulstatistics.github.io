--- title: Fitting a Hurdle Power Law Distribution excerpt: An Rstan program and implementation. ---

This is an example of a Stan model I fit for my research as a graduate assistant. The data is the number (frequency) of people censored in China that were censored zero times, one time, two times,… (count). The task is to find this univariate distribution.

The following code was written directly in Stan because the high level interface Brms doesn’t have the family power law, and it needs to be a hurdle model.

Our model is then

\[\begin{align} \text{HurdlePowerLaw}(n|\alpha,\theta) &= \begin{cases} \theta & \text{if $n=0$} \\ (1-\theta)\text{PowerLaw}(n|\alpha) & \text{if $n\geq1$} \end{cases} \nonumber \\ \text{PowerLaw}(n|\alpha) &= \frac{n^{\alpha}}{\sum_{i=1}^{200}{i^{\alpha}}}, \quad 1 \leq n \leq 200 \nonumber \\ & \alpha \sim \text{Normal}(0,1) \nonumber \\ & \theta \sim \text{Beta}(1,10) \nonumber \end{align}\]

Our code is then:

code <- "
functions {
  //this is the core function
  //it is the power law distribution
  //on 1 to 200
  
  real power_lpmf(int y, real alpha){
    real normalize = 0;
    for (n in 1:200) normalize += pow(n,alpha);
    return alpha * log(y) - log(normalize);
  }
  
  //And this function applies the hurdle model to it
  real hurdle_power_lpmf(int y, real alpha, real theta) { 
    if (y == 0) { 
      return bernoulli_lpmf(1 | theta);
    } else { 
      return bernoulli_lpmf(0 | theta) +  
             power_lpmf(y | alpha);
    } 
  }
}

data {
  int<lower=0> N;  //number of points
  int<lower=0> f[N]; //frequency
  int<lower=0> c[N]; //count
}

transformed data {
  int fsum = sum(f); //for generating random numbers
}

parameters {
  real <upper=0> alpha;
  real <lower=0,upper=1> theta;
}

model {
  // priors. We know the probability of zero is high.
  // And we know alpha should be close to zero.
  // We use a half normal prior from -infty to 0.
  
  theta ~ beta(10, 1);
  alpha ~ normal(0,1) T[,0]; // T sets support.

  // for the model itself, each value for count is weighed by 
  // the frequency for that count. This is a sufficient statistic.
  
  for(n in 1:N) {
    target += f[n] * hurdle_power_lpmf(c[n]|alpha, theta);
  }
}

generated quantities{
  // for model testing we want a  consider each value count a
  // binomial experiment with hurdle power probability.
  // this is an easy way to get a sample from a discrete
  // custom variable.
  
  int freq_test[N];
  for(n in 1:N) {
    freq_test[n] = binomial_rng(fsum,exp(hurdle_power_lpmf(c[n]|alpha, theta)));
  }
}
"

data = list(N=nrow(user),f=user$freq,c=user$count)

fit = stan(model_code=code,data=data)
saveRDS(fit, "power.rds")

All looks good, we have nice normal looking posteriors for the model parameters. And the log probability shows a nice skewed left distribtion. This is because the Hamiltonian Sampler Stan uses gets closer and closer to the best fitting parameters.

Our model visually fits pretty good. So for our model, we first get the constant for the power law distribution. And we also invert it to put it on the bottom of the fraction.

alpha <- mean(extract(fit,par='alpha')[[1]])
theta <- mean(extract(fit,par='theta')[[1]])

support <- 1:200
constant <- (1-theta)/sum(support^alpha)
constant <- 1/constant
alpha <- round(alpha,3)
theta <- round(theta,5)
constant <- round(constant,3)

Our final model is then:

\[\begin{align} \text{HurdlePowerLaw}(n) &= \begin{cases} 0.99915, & \text{if $n=0$} \\ \frac{n^{-1.439}}{3140.867}, & \text{if $1 \leq n \leq 200$} \nonumber \end{cases} \end{align}\]

The caveat for this is also that the variance around this line increases as count increases. But our model appears to be mostly unbiased, at the end there appear to be more points below the line, and in the middle there appear to be more points above the line, so it isn’t completely unbiased.

We could go on to model the increase in variance. We could also do some other model cheking, but the brunt of the model looks to be good. We could also make some simpler models and compare WAIC’s, get a pseudo-R squared, etc.
